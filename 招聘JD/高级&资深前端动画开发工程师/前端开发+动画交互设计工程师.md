# 面试题

## 第一部分：数学功底与渲染底层

---

### 第一题：LERP（线性插值）平滑跟随

**题目：**  
实操或口述：如何实现一个带“阻尼感”的鼠标跟随效果（例如 Apple 官网常见的平滑圆形光标）？

- **看似考察：**  
  - 简单的鼠标事件监听

- **实际考察：**  
  - **数学基础：** 是否掌握核心公式 `$current + (target - current) \times easing$`
  - **渲染机制：** 是否知道必须使用 `requestAnimationFrame` 保证平滑，而非 `setTimeout`
  - **性能优化：** 是否强制使用 `transform: translate3d` 开启 GPU 加速，避免触发布局重排（Reflow）
  - **代码习惯：** 在 React 组件销毁时是否知道取消动画帧（`cancelAnimationFrame`）以防内存泄漏

---

### 第二题：CSS 渲染流水线的“避坑”指南 —— 【基础·初级】

**题目：**  
如果一个动画在低端设备上掉帧，你发现它正在修改 top/left 或 width/height，你会建议改成什么？为什么？

- **看似考察：**  
  - 属性记忆

- **实际考察：**  
  - **渲染原理：** 是否理解浏览器渲染流水线（Layout -> Paint -> Composite）
  - **性能瓶颈：** 明确 left/top 会触发“重排”，而 transform/opacity 仅在“合成层”处理，不占用主线程计算
  - **分层意识：** 是否知道 `will-change` 属性的使用场景及其负面影响（内存消耗）

---

### 第三题：弹簧物理 (Spring Physics) 的质感 —— 【进阶·高级】

**题目：**  
Apple 的交互非常“弹性”，这种手感与传统的 ease-in-out 贝塞尔曲线动画在逻辑实现上有何本质区别？

- **看似考察：**  
  - 审美感知

- **实际考察：**  
  - **物理模型：** 贝塞尔曲线基于“固定时间”，而弹簧动画基于物理参数（质量 Mass、张力 Tension、摩擦力 Friction）
  - **交互中断：** 弹簧动画支持“动量连续”，即动画在执行中被用户再次触发时，速度会叠加而非瞬间重置（Interruption）
  - **工具掌握：** 是否使用过 Framer Motion 的 `type: "spring"` 或 GSAP 的 `CustomEase`

---

### 第四题：遮罩放大转场 (Clip-Path Reveal) —— 【进阶·高级】

**题目：**  
点击按钮后，一个圆形遮罩以点击位置为中心放大至铺满全屏，呈现新页面。如何计算这个圆的最终半径？

- **看似考察：**  
  - 属性调用

- **实际考察：**  
  - **空间几何：** 考察勾股定理。需计算点击坐标到屏幕四个顶点（Top-Left, Top-Right 等）的最远距离
  - **坐标转换：** 如何获取相对于视口的点击坐标（clientX/Y）并转换为 SVG 或容器内的坐标空间
  - **边缘处理：** 动画结束时如何优雅地移除 clip-path 以减少长期的渲染压力

## 第二部分：滚动叙事与专项性能

---

### 第五题：滚动序列帧的极致调优 (Canvas vs Video) —— 【核心·高级】

**题目：**  
实现 Apple 风格的产品随滚动拆解效果，你会选视频驱动还是 Canvas 图片序列？如何处理 100 张高清图的并发加载？

- **看似考察：**  
  - 技术选型

- **实际考察：**  
  - **多端限制：** 熟知 iOS Safari 在低电量模式下会强制暂停视频滚动控制，导致动画卡死
  - **内存管理：** 100 张图片如何做预加载队列？是否知道在 Canvas 渲染后及时释放不再需要的 Blob URL
  - **帧映射算法：** 滚动像素（ScrollTop）到帧索引（Frame Index）的线性映射及取整处理

---

### 第六题：SVG 路径动画的“黑科技” —— 【基础·中级】

**题目：**  
品牌 Logo 路径要求随滚动像被笔画出来一样。核心属性是什么？如果路径极其复杂，如何获取总长度？

- **看似考察：**  
  - 属性记忆

- **实际考察：**  
  - **核心原理：** `stroke-dasharray`（虚线长度）和 `stroke-dashoffset`（偏移量）的配合
  - **API 掌握：** 是否知道原生 `path.getTotalLength()`
  - **实战经验：** 如何处理 SVG 导出时的原始坐标偏移？如何利用 GSAP drawSVG 插件简化复杂的描边逻辑

---

### 第七题：平滑滚动与原生 Sticky 的“战争” —— 【进阶·高级】

**题目：**  
引入 Lenis 或 Locomotive Scroll 这种虚拟滚动插件后，原生的 position: sticky 失效了，为什么？怎么解？

- **看似考察：**  
  - 插件用法

- **实际考察：**  
  - **滚动机制：** 理解虚拟滚动是通过 CSS transform 位移整个容器，原生滚动条并未真正移动，导致 sticky 监听不到滚动位移
  - **方案落地：** 考察是否知道监听插件的回调函数手动更新元素位移，或者通过特殊的 CSS 配置强制恢复

---

### 第八题：滚动监听的性能瓶颈 —— 【核心·中级】

**题目：**  
页面滚动时需要动态更新 50 个元素的阴影和位置。你会如何避免页面滚动时的“粘滞感”？

- **看似考察：**  
  - 防抖节流

- **实际考察：**  
  - **Passive Listeners：** 设置 `{ passive: true }` 避免 JS 阻塞原生滚动响应
  - **Intersection Observer：** 是否知道用该 API 替代滚动监听，只处理视口内的元素，实现自动启停
  - **分层策略：** 为这些元素开启独立的合成层，减少 Paint 的次数

### 第九题：Next.js SSR 动效闪烁 (FOUC) 治理 —— 【核心·中级】

**题目：**  
在 SSR 页面，动画初始状态（如 opacity: 0）在客户端 JS 加载前会闪现出 100% 的原貌，如何彻底根治？

- **看似考察：**  
  - 一个小 Bug

- **实际考察：**  
  - **水合机制 (Hydration)：** 理解 HTML 服务端下发与客户端激活之间的时间差
  - **渲染生命周期：** `useLayoutEffect`（渲染前同步执行）与 `useEffect`（渲染后异步执行）的选型
  - **CSS 方案：** 是否懂得在 HTML 头部利用内联 CSS 或 CSS 变量预设状态，而非依赖 GSAP 初始化
