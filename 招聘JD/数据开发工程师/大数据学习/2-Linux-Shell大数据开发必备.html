二、文件权限与用户管理 安全基础
rwx 权限含义 必背
Linux 中每个文件/目录都有三组权限：所有者(u)、所属组(g)、其他用户(o)。每组包含读(r)、写(w)、执行(x) 三种权限。
权限字母数字对文件对目录
读r4查看文件内容列出目录内容 (ls)
写w2修改文件内容在目录中创建/删除文件
执行x1运行脚本/程序进入目录 (cd)
示例：-rwxr-xr-- 解读 所有者：rwx (4+2+1=7) → 读写执行
所属组：r-x (4+0+1=5) → 读和执行
其他人：r-- (4+0+0=4) → 只读
数字表示：754
权限管理命令
  • chmod 755 script.sh：设置权限为 rwxr-xr-x
  • chmod +x script.sh：给所有人添加执行权限
  • chmod u+w,g-x file：所有者加写权限、组去掉执行权限
  • chown hadoop:hadoop file：修改文件所有者和所属组
  • chown -R hadoop:hadoop /data/：递归修改目录所有者
  • chgrp devteam file：仅修改所属组
常见权限数字 755：目录/脚本（所有者全部权限，其他人可读可执行）
644：普通文件（所有者可读写，其他人只读）
600：敏感文件（仅所有者可读写，如 SSH 私钥）
777：所有权限全开（极度不安全，生产环境禁用）
sudo 与 root 权限
  • root：超级管理员，拥有系统一切权限，生产环境不建议直接登录 root
  • sudo：普通用户临时获取 root 权限执行单条命令
  • sudo su -：切换到 root 用户
  • /etc/sudoers：配置哪些用户可以使用 sudo（通过 visudo 编辑）
  • sudo -u hadoop hdfs dfs -ls /：以 hadoop 用户身份执行命令
大数据场景：HDFS 权限 vs Linux 权限 进阶
维度Linux 文件权限HDFS 文件权限
存储位置本地磁盘文件系统HDFS 分布式文件系统
权限模型标准 POSIX rwx模仿 POSIX 但不完全一致
用户验证系统 /etc/passwd默认信任客户端用户名（不安全）
权限命令chmod / chownhdfs dfs -chmod / -chown
企业安全PAM / LDAPKerberos + Ranger 精细权限控制
注意：HDFS 默认不强制检查权限（dfs.permissions.enabled=false 时），
生产集群必须开启权限检查并配合 Kerberos 认证

三、文本处理三剑客（grep / awk / sed） 日志分析利器
grep — 文本搜索 必背
在文件中搜索匹配正则表达式的行，是日志排查的第一选择。
参数说明示例
-i忽略大小写grep -i "error" app.log
-r递归搜索目录grep -r "NullPointer" logs/
-n显示行号grep -n "WARN" app.log
-v反向匹配（排除）grep -v "DEBUG" app.log
-c统计匹配行数grep -c "ERROR" app.log
-A/-B/-C显示匹配行的上下文grep -A 5 "Exception" app.log
-E扩展正则（等同 egrep）grep -E "ERROR|FATAL" app.log
实用组合：统计每小时错误数量 grep "ERROR" app.log | awk '{print $2}' | cut -c1-13 | sort | uniq -c
awk — 列处理利器 必背
按行读取文件，自动按分隔符切分为字段（$1, $2, $3...），支持条件过滤和聚合计算。
基本语法 awk -F'分隔符' '条件 {动作}' 文件名
  • $0：整行内容；$1：第一列；$NF：最后一列
  • NR：当前行号；NF：当前行字段数
  • -F','：指定逗号为分隔符（默认空格/Tab）
  • 条件过滤：awk '$3 > 100 {print $1, $3}' data.csv
  • 聚合统计：awk '{sum+=$3} END {print sum}' data.csv
实战：分析 Spark 日志中各 Stage 耗时 grep "finished in" spark.log | awk '{print $4, $NF}' | sort -t' ' -k2 -nr | head -10
sed — 流式文本编辑
对文本进行替换、删除、插入操作，支持正则表达式，可直接修改文件。
  • 替换：sed 's/old/new/g' file（g = 全局替换）
  • 删除行：sed '/pattern/d' file（删除匹配行）
  • 删除空行：sed '/^$/d' file
  • 插入行：sed '3i\插入的文本' file（第3行前插入）
  • 只看指定行：sed -n '10,20p' file（打印10-20行）
  • 直接修改文件：sed -i 's/old/new/g' file（-i 原地修改）
批量修改配置文件 sed -i 's/spark.executor.memory=2g/spark.executor.memory=4g/g' spark-defaults.conf
管道串联与典型场景 核心
管道 | 将前一个命令的输出作为下一个命令的输入，是 Linux 最强大的组合能力。
场景1：从 Spark 日志提取错误信息并去重统计 grep "ERROR" spark-executor.log \
  | awk -F':' '{print $4}' \
  | sort | uniq -c | sort -rn | head -20
场景2：统计 Nginx 日志中接口调用频率 TOP10 awk '{print $7}' access.log \
  | grep -v "\.(css|js|png)" \
  | sort | uniq -c | sort -rn | head -10
场景3：查找占用磁盘最大的日志文件 find /var/log -name "*.log" -exec du -sh {} \; | sort -rh | head -10

四、Shell 脚本基础 自动化运维第一步
变量定义与引号区别 必背
  • 变量赋值：NAME="hadoop"（等号两边不能有空格）
  • 使用变量：echo $NAME 或 echo ${NAME}
  • 双引号 ""：会解析变量和命令替换，echo "user is $NAME"
  • 单引号 ''：原样输出，不解析变量，echo 'user is $NAME'
  • 反引号 ``：执行命令并返回结果，TODAY=`date +%Y%m%d`
  • $(...) ：等同反引号但可嵌套，TODAY=$(date +%Y%m%d)
常用内置变量 $HOME（家目录） $USER（当前用户） $PWD（当前路径）
$PATH（命令搜索路径） $SHELL（当前使用的Shell）
条件判断 if / elif / else
基本语法 #!/bin/bash
if [ -f "/data/input.csv" ]; then
  echo "文件存在，开始处理"
elif [ -d "/data/" ]; then
  echo "目录存在但文件不存在"
else
  echo "目录和文件都不存在"
fi
  • -f file：文件是否存在且为普通文件
  • -d dir：目录是否存在
  • -z "$var"：变量是否为空字符串
  • -n "$var"：变量是否非空
  • 数值比较：-eq（等于） -ne（不等） -gt（大于） -lt（小于）
  • 字符串比较：= 和 !=（在 [ ] 中使用）
循环 for / while
for 循环：遍历列表 for node in master slave1 slave2 slave3; do
  echo "正在同步配置到 $node ..."
  scp hadoop-env.sh $node:/opt/hadoop/etc/
done
for 循环：遍历数字范围 for i in $(seq 1 10); do
  echo "处理分区 partition_$i"
done
while 循环：逐行读取文件 while IFS= read -r line; do
  echo "处理: $line"
done < node_list.txt
函数与命令行参数
函数定义与调用 check_hdfs_path() {
  local path=$1
  if hdfs dfs -test -d "$path"; then
    echo "路径存在: $path"
    return 0
  else
    echo "路径不存在: $path"
    return 1
  fi
}

check_hdfs_path "/user/hive/warehouse/orders"
  • $1 $2 $3：第 1、2、3 个参数
  • $#：参数个数
  • $@：所有参数（作为独立字符串列表）
  • $*：所有参数（作为一个字符串）
  • $?：上一条命令的退出码（0=成功，非0=失败）
  • set -e：脚本中任何命令失败立即退出（推荐开启）
实用示例：自动备份 Hive 表数据脚本 实战
backup_hive_table.sh #!/bin/bash
set -e

TABLE_NAME=${1:?"用法: $0 <表名> [日期]"}
DT=${2:-$(date -d "yesterday" +%Y-%m-%d)}
BACKUP_DIR="/user/backup/hive/${TABLE_NAME}/${DT}"
SOURCE_DIR="/user/hive/warehouse/${TABLE_NAME}/dt=${DT}"

echo "[$(date)] 开始备份: ${TABLE_NAME}, 分区: ${DT}"

if ! hdfs dfs -test -d "${SOURCE_DIR}"; then
  echo "源路径不存在: ${SOURCE_DIR}，跳过"
  exit 0
fi

hdfs dfs -mkdir -p "${BACKUP_DIR}"
hdfs dfs -cp "${SOURCE_DIR}/*" "${BACKUP_DIR}/"

FILE_COUNT=$(hdfs dfs -ls "${BACKUP_DIR}" | wc -l)
echo "[$(date)] 备份完成，共 ${FILE_COUNT} 个文件 → ${BACKUP_DIR}"
调用方式 chmod +x backup_hive_table.sh
./backup_hive_table.sh ods_order_detail 2025-03-10

五、系统资源监控 集群运维必备
CPU 与内存监控 核心
  • top：实时查看 CPU / 内存 / 进程，按 P 按 CPU 排序、按 M 按内存排序、按 q 退出
  • htop：top 的增强版，支持鼠标操作，可视化 CPU 核心使用率（需安装 yum install htop）
  • free -h：查看内存总量、已用、可用、缓存，-h 以人类可读格式显示
  • vmstat 1 5：每秒采样一次共5次，查看 CPU/内存/IO/swap 综合状态
free -h 输出关键列 total（总量） used（已用） free（空闲） buff/cache（缓存）
available（可用）= free + 可回收的 buff/cache
磁盘空间监控
  • df -h：查看各挂载点磁盘使用情况（-h 人类可读）
  • df -i：查看 inode 使用率（大量小文件会耗尽 inode）
  • du -sh *：查看当前目录下每个文件/子目录的大小
  • du -sh /data/*：查看 /data 下各子目录大小
  • du -sh /data/ --max-depth=1：只显示第一层目录大小
快速找出大文件 find / -type f -size +1G -exec ls -lh {} \; 2>/dev/null
进程管理
  • ps aux：查看所有进程的详细信息（CPU%、MEM%、PID）
  • ps aux | grep spark：筛选 Spark 相关进程
  • kill PID：发送 SIGTERM 信号（优雅终止）
  • kill -9 PID：发送 SIGKILL 信号（强制终止）
  • nohup command &：后台运行且终端关闭不中断
  • jobs / fg / bg：查看/切换前后台任务
网络与端口 + Java 进程
  • netstat -tlnp：查看监听中的 TCP 端口及对应进程
  • ss -tlnp：netstat 的替代工具，更快（推荐使用）
  • lsof -i :8080：查看占用 8080 端口的进程
  • jps：查看当前运行的 Java 进程（Hadoop 生态必备）
大数据常见 Java 进程（jps 输出） NameNode / DataNode / ResourceManager / NodeManager
HiveServer2 / HiveMetaStore
Kafka / QuorumPeerMain (ZooKeeper)
SparkSubmit / CoarseGrainedExecutorBackend
快速检查集群关键进程是否存活 for node in master slave1 slave2; do
  echo "=== $node ==="; ssh $node "jps"
done

Crontab vs 企业级调度系统 进阶
维度CrontabAirflow / DolphinScheduler
任务依赖不支持任务间依赖，只能通过脚本手动控制原生 DAG 依赖管理，上游成功才执行下游
失败重试不支持，需自己在脚本中实现内置重试策略，支持指定重试次数和间隔
告警通知无（需自己写邮件/钉钉通知脚本）内置邮件/企微/钉钉/Slack 告警
可视化纯命令行，不直观Web UI 管理，任务状态一目了然
分布式单机运行支持 Worker 集群分布式执行
适用场景简单脚本、开发测试、个人任务生产 ETL 流水线、复杂依赖调度
生产建议：开发测试可用 Crontab 快速验证，正式上线后迁移到 Airflow / DolphinScheduler 管理

SSH 远程登录与免密配置 必背
  • ssh user@host：远程登录服务器
  • ssh -p 2222 user@host：指定端口登录
  • ssh user@host "command"：远程执行命令后返回
免密登录配置（大数据集群必备） # 1. 生成密钥对（一路回车）
ssh-keygen -t rsa

# 2. 将公钥发送到目标机器
ssh-copy-id hadoop@slave1
ssh-copy-id hadoop@slave2
ssh-copy-id hadoop@slave3

# 3. 验证免密登录
ssh slave1 # 不再需要输入密码

文件传输：scp / rsync
  • scp file user@host:/path/：上传文件到远程服务器
  • scp user@host:/path/file .：从远程下载文件到本地
  • scp -r dir/ user@host:/path/：递归传输目录
  • rsync -avz dir/ user@host:/path/：增量同步（只传变化部分，更高效）
  • rsync --delete：同步并删除目标端多余的文件
scp vs rsync scp：全量传输，简单直接
rsync：增量传输，断点续传，大文件/大目录首选
curl / wget 下载
  • curl -O url：下载文件并保留原始文件名
  • curl -o name url：下载并指定保存文件名
  • curl -X POST -d 'data' url：发送 POST 请求
  • wget url：下载文件（支持断点续传 -c）
  • wget -r -l 2 url：递归下载，深度2层
