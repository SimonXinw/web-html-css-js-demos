一、流量来源分类与分析 理解用户从哪里来
六大流量来源渠道 必背
  • Organic Search（自然搜索）：Google / Bing 搜索结果中非广告位的点击，依赖 SEO 排名
  • Paid Search（付费搜索）：Google Ads 搜索广告点击，按 CPC 计费
  • Social（社交媒体）：Facebook / Instagram / TikTok / X 自然帖文与付费广告
  • Email（邮件营销）：Newsletter、促销邮件、弃购挽回邮件，ROI 通常最高
  • Direct（直接访问）：用户直接输入网址或从书签进入，代表品牌认知
  • Referral（引荐流量）：从其他网站链接跳转，包括联盟营销（Affiliate）、博客引用
流量质量评估维度
渠道典型 CVR成本用户质量
Organic Search2%~5%低（长期投入）高意图，主动搜索
Paid Search2%~4%中高（CPC 竞价）高意图，精准关键词
Social Paid1%~3%中（CPM/CPC）中，兴趣驱动
Email3%~8%极低高，已有用户基础
Direct3%~6%品牌积累最高，忠实用户
Referral1%~4%佣金/免费取决于来源站质量
流量漏斗全链路 核心
广告曝光
Impression → CTR 1%~3%
点击访问
Click / UV → 跳出率 40%~60%
浏览商品
Product View → 加购率 8%~15%
加入购物车
Add to Cart → 结账率 50%~70%
完成支付
Purchase → 端到端 CVR 1%~3%
分析要点：找到漏斗中跌幅最大的环节，集中优化。跳出率高 → 改落地页；加购后流失 → 检查结账流程
付费流量 vs 自然流量策略
健康的流量结构应该是付费流量拉新 + 自然流量承接的组合。
  • 早期阶段：付费占比高（70%+），快速测试产品市场匹配度
  • 增长阶段：逐步提升 SEO / Email / Social 自然流量，降低对付费的依赖
  • 成熟阶段：自然 + 回访流量占比 50%+ 为健康标志
  • 风险信号：付费流量占比 > 80% → 一旦停投广告，收入断崖式下降
理想流量组合 = 30% Paid + 25% Organic + 20% Direct + 15% Email + 10% Social/Referral

二、UTM 参数追踪体系 精确追踪每一分钱的来源
UTM 五大参数详解 必背
参数含义示例值是否必填
utm_source流量来源平台google / facebook / newsletter必填
utm_medium营销媒介类型cpc / email / organic_social必填
utm_campaign活动/计划名称bfcm_2025 / spring_sale必填
utm_content区分素材/广告变体video_v2 / banner_red可选
utm_term付费搜索关键词running+shoes / cheap+bags可选
UTM 命名规范 核心
团队统一的命名规范是 UTM 有效的前提，否则数据一片混乱。
  • 全部小写：GA 区分大小写，facebook ≠ Facebook ≠ FACEBOOK
  • 下划线分隔：用 _ 代替空格和特殊字符，如 spring_sale_2025
  • 固定枚举值：source / medium 用固定字典（google、facebook、email…）
  • 有意义的 campaign 名：包含活动 + 日期，如 bfcm_nov_2025
  • content 区分素材：video_15s_v2 / image_carousel_blue
完整 UTM URL 示例 https://shop.com/products/shoes
?utm_source=facebook
&utm_medium=paid_social
&utm_campaign=bfcm_nov_2025
&utm_content=video_15s_sneaker
&utm_term=running_shoes
UTM 数据在 GA4 中的呈现
用户点击带 UTM 的链接 → GA4 自动解析参数 → 写入 Session Source/Medium → 报表按渠道聚合
  • 流量获取报告：Source / Medium 维度展示各渠道的会话数、转化率
  • 用户获取报告：展示首次访问来源，追踪新用户的获取渠道
  • 事件报告：关联 UTM 参数与 purchase / add_to_cart 等关键事件
GA4 维度映射：utm_source → Session source，utm_medium → Session medium，utm_campaign → Session campaign
常见 UTM 错误与排查
  • 参数拼写错误：utm_souce（少了 r）→ GA 无法识别，归入 (direct)
  • 大小写不统一：Facebook 和 facebook 被视为两个来源，数据分散
  • 重复参数：URL 中出现两次 utm_source，GA 取最后一个
  • 落地页重定向丢参数：301 跳转未携带 query string → UTM 丢失
  • 内部链接加 UTM：站内链接不应带 UTM，否则会覆盖真实外部来源
排查工具：GA4 DebugView 实时检查事件参数 / Chrome 扩展 UTM.io 自动生成标准 URL

三、SEO 搜索引擎优化基础 免费流量的长期投资
SEO 三大核心要素 必背
  • 技术 SEO (Technical)：网站能否被搜索引擎正确抓取与索引（爬虫可达、站点地图、HTTPS、速度）
  • 内容 SEO (On-Page)：页面内容是否匹配用户搜索意图（关键词、标题、Meta Description、内链）
  • 权威 SEO (Off-Page)：外部网站对你的信任度（外链数量与质量、品牌提及、社交信号）
Google 排名 ≈ 内容相关性 × 页面权威度 × 用户体验信号（Core Web Vitals）
关键词研究流程
确定种子词 → 工具拓展长尾词 → 分析搜索量/难度 → 判断搜索意图 → 匹配落地页
  • 搜索意图分类：信息型（how to…）/ 导航型（brand name）/ 交易型（buy…）/ 商业调查型（best…）
  • 常用工具：Ahrefs / SEMrush / Google Keyword Planner / Ubersuggest
  • 长尾关键词：搜索量小但精准，竞争低，适合新站快速获取排名
Core Web Vitals 核心指标
指标含义良好阈值优化方向
LCP最大内容绘制时间≤ 2.5s优化图片、CDN、服务端响应
INP交互到下一次绘制延迟≤ 200ms减少 JS 阻塞、代码拆分
CLS累积布局偏移≤ 0.1图片预留尺寸、字体预加载
Core Web Vitals 直接影响 Google 搜索排名，是技术 SEO 的硬指标。PageSpeed Insights 可检测
SEO 效果数据指标
  • Organic Traffic：自然搜索带来的会话数，核心增长指标
  • Keyword Rankings：目标关键词在搜索结果中的位置（Top 3 / Top 10）
  • Click-Through Rate：搜索展示后的点击率，与标题和描述质量相关
  • Indexed Pages：被 Google 收录的页面数量
  • Backlinks：外链数量与域名权重（Domain Rating / Authority）
  • Organic Conversions：自然搜索流量带来的转化数及金额
数据来源：Google Search Console（免费）+ Ahrefs/SEMrush（付费）+ GA4（转化追踪）

四、Google Ads 广告投放 搜索广告 + 购物广告
Google Ads 广告类型 必背
类型展示位置适用场景计费方式
搜索广告 SearchGoogle 搜索结果页顶部高购买意图关键词CPC
购物广告 Shopping搜索结果的商品卡片电商核心，展示图片+价格CPC
展示广告 Display合作网站的广告位品牌曝光、再营销CPM / CPC
视频广告 VideoYouTube 片前/片中品牌故事、产品演示CPV / CPM
效果最大化 PMax跨所有 Google 渠道自动化全渠道投放CPA 目标
Google Ads 核心指标 核心
CPC
单次点击成本
CPC = 花费 / 点击数
CTR
点击率
CTR = 点击 / 展示 × 100%
CPA
单次转化成本
CPA = 花费 / 转化数
QS
质量得分 (1-10)
QS = 相关性 + CTR + 落地页体验
Google Ads 竞价公式 Ad Rank = 出价 (Bid) × 质量得分 (QS)
实际 CPC = 下一名 Ad Rank / 你的 QS + $0.01
→ QS 越高，用更低价格获得更高排名
搜索广告账户结构
Account（账户） → Campaign（广告系列） → Ad Group（广告组） → Keywords + Ads（关键词+广告）
  • Campaign 层：设置预算、地理位置、投放时间、出价策略
  • Ad Group 层：按主题/产品类别分组关键词，每组 10~20 个相关词
  • 关键词匹配类型：广泛匹配 / 词组匹配 / 精确匹配 / 否定关键词
  • RSA 响应式广告：提供多条标题和描述，Google 自动组合最优版本
Google Shopping 广告优化
  • Merchant Center：产品数据源（Feed）管理中心，商品信息必须准确、及时更新
  • Feed 关键字段：title / description / price / image_link / availability / gtin
  • 标题优化：品牌 + 产品类型 + 核心属性（颜色/材质/尺寸），搜索匹配更精准
  • 定价竞争力：Google 会对比竞品价格，价格有优势的产品展示频率更高
Shopping 广告不设关键词，Google 根据 Feed 中的商品标题和描述自动匹配搜索词
→ Feed 质量 = Shopping 广告效果的基础

五、Facebook / Meta Ads 广告投放 社交媒体广告核心平台
Facebook Ads 账户架构 核心
Business Manager → Ad Account（广告账户） → Campaign（广告系列） → Ad Set（广告组） → Ad（广告）
  • Campaign 层：选择营销目标 — 知名度 / 流量 / 互动 / 线索 / 销量
  • Ad Set 层：定义受众、版位、预算、排期
  • Ad 层：创建素材（图片/视频 + 文案 + CTA 按钮）
受众定向策略
受众类型定义典型用法
核心受众 Core按兴趣/行为/人口统计定向拓新：锁定对竞品或品类有兴趣的用户
自定义受众 Custom基于已有用户数据（网站/APP/CRM）再营销：30天内加购未购买的用户
类似受众 Lookalike系统找到与种子用户相似的新人群拓新：基于高价值购买用户的 1%~5% LAL
Facebook Pixel 是数据回传的核心：安装在网站上，追踪 PageView / AddToCart / Purchase 等事件
→ Pixel 数据越多，广告算法优化越精准
Facebook Ads 关键指标
CPM
千次展示成本
CPM = 花费 / 展示 × 1000
CPC
单次链接点击成本
CPC = 花费 / 链接点击
CTR
链接点击率
CTR = 点击 / 展示 × 100%
ROAS
广告花费回报
ROAS = 购买金额 / 花费
Freq
频次（平均触达次数）
Freq = 展示 / 触达人数
Freq > 3~4 时需更换素材防疲劳
CPA
单次购买成本
CPA = 花费 / 购买次数
电商核心效率指标
广告素材测试方法论
  • 素材为王：Facebook 广告效果 70% 取决于素材质量，30% 取决于受众和出价
  • A/B Test：同一受众下测试不同素材（图片 vs 视频 / 短文案 vs 长文案）
  • Hook 3秒法则：视频前 3 秒决定用户是否继续观看，需要强 Hook
  • UGC 素材：用户生成内容风格的素材，CTR 通常比品牌精修素材高 30%+
  • 素材疲劳监控：CTR 持续下降 + 频次上升 → 该更换新素材

TikTok Ads 广告产品 核心
广告类型展示位置特点适用场景
In-Feed AdsFor You 信息流原生视频形式，支持 CTA电商转化、APP下载
TopView打开APP的第一条视频最大曝光位，日均触达百万品牌大曝光

TikTok vs Facebook 投放差异
维度TikTok AdsFacebook Ads
素材形式竖屏短视频为主（9:16）图片+视频均可，版位多样

TikTok 素材创作原则
  • "不像广告的广告"：最佳素材看起来像用户自发创作的内容，而非品牌硬广
  • 前 1 秒 Hook：比 Facebook 更短的注意力窗口，第 1 秒就要抓住用户
  • 趋势结合：利用热门音乐、挑战赛、滤镜等元素提升原生感
  • 真人出镜：真人讲解/开箱/测评类素材完播率和互动率最高
  • 高频迭代：素材生命周期短（7~14天），需持续产出新素材
TikTok 素材公式 = Hook（1s）+ 痛点/场景（3s）+ 产品展示（5s）+ CTA（2s）
总时长控制在 15~30s 效果最佳
TikTok 数据追踪体系
TikTok Pixel（浏览器端） + Events API（服务器端） → 事件去重匹配 → 广告归因
  • 必须事件：ViewContent / AddToCart / InitiateCheckout / CompletePayment
  • 双端上报：Pixel + Events API 配合，避免 iOS 隐私政策导致的数据丢失
  • 归因窗口：默认 7天点击 + 1天浏览，可在广告管理后台调整

七、广告归因模型 功劳归谁？最核心的难题
归因模型全景对比 必背
归因模型功劳分配方式优点缺点
Last Click100% 给最后一次点击简单直接，易于执行忽略上游渠道贡献
First Click100% 给首次接触重视用户发现渠道忽略转化促成渠道
Linear均分给所有触点公平看待每个渠道无法区分重要性差异
Time Decay越接近转化的触点分配越多平衡上下游可能低估品牌触点
Position Based首末各 40%，中间均分 20%重视首尾触点中间环节被低估
Data-DrivenML 模型按实际贡献分配最准确，基于真实数据需要大量数据
用户转化路径示例
一个典型的跨境电商用户购买路径：
Day 1: TikTok 视频广告 → 首次认知品牌
Day 3: Google 搜索品牌词 → 主动搜索了解
Day 5: Facebook 再营销广告 → 加入购物车
Day 7: Email 弃购挽回 → 完成购买 $120
不同归因下的功劳分配 Last Click：Email = $120（其他渠道 = $0）
First Click：TikTok = $120
Linear：各 $30
Position Based：TikTok $48 + Google $12 + Facebook $12 + Email $48
归因窗口（Attribution Window）
  • 点击归因窗口：用户点击广告后 N 天内的转化归因给该广告（通常 7~28 天）
  • 浏览归因窗口：用户看到广告但未点击，N 天内转化也计入（通常 1 天）
  • Facebook 默认：7天点击 + 1天浏览
  • Google Ads 默认：30天点击
  • 窗口越长，归因转化越多：但也越不精确，容易将自然转化计入广告
iOS 14.5+ 限制了浏览归因数据，导致 Facebook 报告的转化数偏低
→ 需要结合服务端追踪（CAPI / Events API）补全数据
Multi-Touch Attribution (MTA) 实践
多触点归因是企业级的归因方案，需要打通各渠道数据。
  • 数据打通：将 GA4 + Facebook + TikTok + Email 数据汇聚到数据仓库
  • 用户拼接：通过 email / phone / user_id 打通跨平台的用户身份
  • 路径分析：还原每个转化用户的完整触点路径
  • 模型训练：用 Shapley Value / Markov Chain 等算法计算各渠道贡献
数据团队的核心产出 输入：各平台广告日志 + 网站行为日志 + 订单数据
输出：每个渠道的真实增量贡献 + 预算分配建议

八、ROI / ROAS 投资回报分析 广告效率的终极衡量标准
ROI vs ROAS 区别 必背
指标公式含义示例
ROI(收入 - 成本) / 成本 × 100%净利润率，考虑所有成本花 $1000，赚 $3000 → ROI = 200%
ROAS广告收入 / 广告花费广告花费回报倍数花 $1000 广告费，收入 $4000 → ROAS = 4
关键区别 ROI 考虑全部成本（产品成本 + 物流 + 广告 + 运营）→ 衡量真实盈利能力
ROAS 只看广告花费与收入的比值 → 衡量广告投放效率
ROAS 高 ≠ 一定赚钱（如果产品成本高、退货率高）
电商盈利公式拆解 核心
单订单利润计算 Revenue（收入）= 商品售价
- COGS（产品成本）
- Shipping（物流费）
- Platform Fee（平台费/支付手续费）
- Ad Cost（分摊广告费 = CPA）
= Net Profit（净利润）
盈亏平衡 ROAS 计算 假设毛利率 = 60%（产品+物流+平台费占收入 40%）
Break-even ROAS = 1 / 毛利率 = 1 / 0.6 = 1.67
→ ROAS > 1.67 才开始盈利，ROAS > 3 为健康水平
各渠道 ROAS 基准参考
渠道典型 ROAS说明
Google Search（品牌词）8~15x用户已认知品牌，转化率最高
Google Shopping4~8x高购买意图，电商核心
Facebook（再营销）5~10x已有兴趣用户，容易转化
Facebook（拓新）2~4x冷流量，成本较高
TikTok Ads2~5x冲动消费属性强，品类差异大
Email Marketing30~50x成本极低，ROI 最高的渠道
LTV : CAC 长期价值分析
单次 ROAS 不能完全评判渠道价值，还需考虑用户长期价值。
核心公式 LTV = AOV × 购买频次 × 客户生命周期
CAC = 总营销花费 / 新用户数
LTV : CAC 比值 > 3 → 业务健康可持续
LTV : CAC 比值 < 1 → 获客越多亏越多
  • 短期亏损可接受：首单 CPA > 首单利润，但 LTV 远超 CAC → 可加大投入
  • Cohort LTV 分析：按获取月份分组，追踪 3/6/12 个月的累计消费
  • 渠道质量差异：Google 搜索获取的用户 LTV 通常 > 社交广告获取的用户

渠道间协同效应
各渠道不是独立运作的，而是相互影响、相互促进的：
  • TikTok → Google：TikTok 曝光后用户主动搜索品牌词，拉动 Google 品牌搜索量
  • Facebook → Email：广告引流到站注册后，Email 进行长期培育和转化
  • SEO → Paid：自然排名和广告同时出现在搜索结果，信任度叠加提高 CTR
  • Retargeting 链条：TikTok 首次触达 → Facebook 再营销 → Email 弃购挽回 → 完成转化
停掉品牌广告后品牌搜索量下降 → 说明付费渠道在为自然流量输血
→ 不能只看单渠道 ROAS，需要看整体预算对全局 Revenue 的影响
预算分配框架
基于漏斗阶段和渠道 ROI 的预算分配方法：
70-20-10 预算分配法则 70% → 已验证有效的渠道（如 Google Shopping、Facebook 再营销）
20% → 有潜力待优化的渠道（如 TikTok 拓新、SEO 内容投入）
10% → 新兴/实验渠道（如 Pinterest Ads、Influencer、播客广告）
  • MER（Marketing Efficiency Ratio）：总收入 / 总营销花费，衡量整体效率
  • 增量测试：停掉某渠道 2 周，观察总收入变化，量化渠道真实贡献
  • 季节性调整：BFCM / Prime Day 等大促期间加大付费预算，日常回归自然增长
再营销（Retargeting）策略分层
受众分层定义策略预期 ROAS
加购未购买7天内加购但未下单展示加购商品 + 限时折扣8~15x
浏览未加购14天内浏览商品页推送热销/新品素材4~8x
老客户复购30~90天前购买过推荐关联商品/新品6~12x
流失用户90天+ 未访问大额优惠券激活2~4x
再营销是 ROAS 最高的广告类型，但受众池有限
→ 必须持续通过拓新广告为再营销"蓄水"

异常诊断分析框架
当 Dashboard 指标异常时，按照以下流程排查：
ROAS 下降排查树 ROAS = Revenue / Ad Spend = (Clicks × CVR × AOV) / (Clicks × CPC)
= CVR × AOV / CPC

CPC 上升？→ 竞争加剧 / 素材疲劳 / 季节性波动
CVR 下降？→ 落地页问题 / 受众质量下降 / 网站故障
AOV 下降？→ 促销折扣力度大 / 低价品占比上升
  • 先排除数据问题：Pixel 是否丢事件？归因窗口是否变更？数据延迟？
  • 再排除外部因素：竞品大促？平台政策变更？季节性因素？
  • 最后定位内部原因：素材衰退？受众重叠？预算分配不合理？
