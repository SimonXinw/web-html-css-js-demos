二、Python 环境安装与配置 数据处理基础
下载与安装 第一步
  • 访问 python.org/downloads 下载 Python 3.10+ 安装包（推荐 3.11 或 3.12）
  • 运行安装程序时，务必勾选 "Add Python to PATH"（底部复选框）
  • 选择 "Customize installation"，确认 pip 被勾选
  • 安装路径建议保持默认，或选择不含中文/空格的路径
验证安装 python --version
pip --version
创建虚拟环境 推荐
虚拟环境隔离不同项目的依赖，避免包版本冲突。每个大数据项目建议单独创建虚拟环境。
创建虚拟环境 python -m venv bigdata-env
激活虚拟环境（PowerShell） bigdata-env\Scripts\Activate.ps1
激活虚拟环境（CMD） bigdata-env\Scripts\activate.bat
退出虚拟环境 deactivate
安装常用包
激活虚拟环境后，安装大数据学习所需的核心包。
一键安装 pip install pandas numpy jupyter duckdb pyspark matplotlib
如果下载慢，使用国内镜像 pip install pandas numpy jupyter duckdb pyspark -i https://pypi.tuna.tsinghua.edu.cn/simple
启动 Jupyter Notebook
Jupyter 是大数据学习的交互式笔记本，适合写代码 + 看结果 + 做笔记。
启动 Jupyter jupyter notebook
  • 运行后自动打开浏览器，访问 http://localhost:8888
  • 新建 Python 3 Notebook 即可开始编写代码
  • Jupyter 会自动使用当前虚拟环境的 Python 解释器
PowerShell 执行策略报错？运行此命令 Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser

三、Java JDK 安装（PySpark 依赖） Spark 底层运行时
下载与安装 JDK 必需
  • 推荐 Adoptium Eclipse Temurin（免费开源 JDK 发行版）
  • 访问 adoptium.net 下载 JDK 8 或 JDK 11 的 Windows x64 MSI 安装包
  • 安装时勾选 "Set JAVA_HOME variable"（安装程序会自动配置环境变量）
  • JDK 8 和 JDK 11 都可以，PySpark 两者都兼容
验证安装 java -version
echo %JAVA_HOME%
手动设置 JAVA_HOME
如果安装时没有自动设置，需要手动配置环境变量。
PowerShell 设置（当前用户永久生效） [System.Environment]::SetEnvironmentVariable("JAVA_HOME", "C:\Program Files\Eclipse Adoptium\jdk-11.0.21.9-hotspot", "User")
CMD 临时设置（仅当前窗口） set JAVA_HOME=C:\Program Files\Eclipse Adoptium\jdk-11.0.21.9-hotspot
  • 也可以通过 系统属性 → 高级 → 环境变量 图形界面设置
  • 路径替换为你的实际安装路径，注意不要包含 \bin
  • 设置后重新打开终端才能生效
常见问题排查
问题表现解决方案
JAVA_HOME 未设置PySpark 启动报错 "JAVA_HOME is not set"按上述步骤设置 JAVA_HOME，重启终端
Java 版本过高PySpark 报 "Unsupported class file version"降级为 JDK 8 或 JDK 11，不要用 JDK 17+
PATH 未包含 Javajava -version 提示命令未找到将 %JAVA_HOME%\bin 添加到系统 PATH

四、PySpark 本地模式安装 在 Windows 跑 Spark
安装 PySpark 核心
pip 安装 pip install pyspark
快速验证 python -c "from pyspark.sql import SparkSession; print('PySpark OK')"
启动交互式 PySpark Shell pyspark
  • PySpark 会自动下载内置的 Spark 二进制文件
  • 本地模式不需要 Hadoop 集群，单机运行即可学习
  • 首次启动较慢，后续会快很多
配置 winutils.exe（Windows 专属）必需
Spark 底层依赖 Hadoop 的某些本地库，在 Windows 上需要 winutils.exe 才能正常运行文件操作。
步骤 1：创建 Hadoop 目录 mkdir C:\hadoop\bin
步骤 2：下载 winutils.exe 访问 github.com/steveloughran/winutils
下载对应 Hadoop 版本的 winutils.exe
放到 C:\hadoop\bin\ 目录下
步骤 3：设置环境变量（PowerShell） [System.Environment]::SetEnvironmentVariable("HADOOP_HOME", "C:\hadoop", "User")
步骤 4：将 bin 加入 PATH 将 C:\hadoop\bin 添加到系统 PATH 环境变量
完整验证：创建 DataFrame 并执行 Spark SQL
将以下代码保存为 test_spark.py，运行 python test_spark.py 验证 PySpark 安装是否完整。
test_spark.py — 完整验证脚本 from pyspark.sql import SparkSession
from pyspark.sql.functions import col, sum as spark_sum

spark = SparkSession.builder \
    .master("local[*]") \
    .appName("WindowsTest") \
    .getOrCreate()

data = [
    ("Alice", "北京", 100),
    ("Bob", "上海", 200),
    ("Alice", "北京", 150),
    ("Charlie", "广州", 300),
    ("Bob", "上海", 250),
]

df = spark.createDataFrame(data, ["name", "city", "amount"])

# DataFrame API
print("=== DataFrame API ===")
df.groupBy("name").agg(spark_sum("amount").alias("total")).show()

# Spark SQL
df.createOrReplaceTempView("orders")
print("=== Spark SQL ===")
spark.sql("""
    SELECT city, COUNT(*) AS order_count, SUM(amount) AS total
    FROM orders
    GROUP BY city
    ORDER BY total DESC
""").show()

spark.stop()
print("PySpark 验证通过！")

五、DuckDB 安装（零配置 SQL 练习） 列式分析引擎
安装与基本使用 简单
安装 pip install duckdb
Python 中使用 import duckdb
result = duckdb.sql("SELECT 42 AS answer")
print(result)
创建本地数据库文件 import duckdb
con = duckdb.connect("mydb.duckdb")
con.sql("CREATE TABLE users (id INT, name VARCHAR)")
con.sql("INSERT INTO users VALUES (1, 'Alice'), (2, 'Bob')")
con.sql("SELECT * FROM users").show()
为什么选 DuckDB？
特性说明
零配置pip install 即用，无需启动服务器进程
列式存储天生适合 OLAP 分析查询，速度远超 SQLite
完整 SQL支持窗口函数、CTE、JOIN、子查询等高级语法
直读文件可直接查询 CSV / Parquet / JSON 文件
Pandas 集成DataFrame 与 DuckDB 表无缝互转
嵌入式进程内运行，无网络开销，适合单机学习
快速验证示例：窗口函数 + CSV 读取
直接查询 CSV 文件 import duckdb

# 直接对 CSV 文件执行 SQL（不需要建表）
duckdb.sql("SELECT * FROM read_csv_auto('sales.csv') LIMIT 10").show()

# 窗口函数练习
duckdb.sql("""
    SELECT
        product,
        month,
        revenue,
        SUM(revenue) OVER (PARTITION BY product ORDER BY month) AS cumulative_rev,
        ROW_NUMBER() OVER (PARTITION BY product ORDER BY revenue DESC) AS rank
    FROM read_csv_auto('sales.csv')
""").show()

# 与 Pandas 互操作
import pandas as pd
df = pd.DataFrame({"x": [1,2,3], "y": [10,20,30]})
duckdb.sql("SELECT x, y, x * y AS product FROM df").show()

六、Docker Desktop 安装与配置 容器化大数据组件
前置条件 必须确认
  • Windows 10 (2004+) 或 Windows 11，家庭版和专业版均可
  • 需要开启 WSL2 或 Hyper-V（Docker Desktop 安装时会引导开启）
  • BIOS 中开启 虚拟化（VT-x / AMD-V）
  • 至少 8GB 内存（分配给 Docker 至少 4GB）
检查虚拟化是否开启（PowerShell） systeminfo | findstr /i "虚拟化"
如果显示 "已启用固件中的虚拟化: 是" 则 OK 如果显示"否"，需进入 BIOS 开启 Intel VT-x 或 AMD-V
下载安装 Docker Desktop
  • 访问 docker.com/products/docker-desktop 下载 Windows 版
  • 运行安装程序，推荐选择 "Use WSL 2 instead of Hyper-V"
  • 安装完成后重启电脑
  • 首次启动 Docker Desktop，等待引擎初始化完成
验证安装 docker --version
docker run hello-world
Docker Compose（已内置） docker compose version
资源配置建议
打开 Docker Desktop → Settings → Resources，调整分配给 Docker 的资源。
资源最低配置推荐配置
CPU2 核4 核
内存4 GB8 GB
磁盘32 GB64 GB
Swap1 GB2 GB
常见问题排查
问题解决方案
Hyper-V 未启用PowerShell 管理员运行：
Enable-WindowsOptionalFeature -Online -FeatureName Microsoft-Hyper-V -All

镜像拉取速度慢Docker Desktop → Settings → Docker Engine，添加国内镜像加速器
启动失败 / 闪退检查 BIOS 虚拟化、Windows 更新、重装 Docker Desktop
配置镜像加速器（Docker Engine JSON） {

七、WSL2 安装（Windows 上的 Linux） 可选但强烈推荐
一键安装 WSL2 简单
WSL2（Windows Subsystem for Linux 2）让你在 Windows 内运行完整的 Linux 环境，无需虚拟机。
以管理员身份运行 PowerShell wsl --install
  • 默认安装 Ubuntu 发行版
  • 安装完成后需要重启电脑
  • 重启后自动弹出 Ubuntu 窗口，设置用户名和密码
选择其他发行版 wsl --list --online
wsl --install -d Ubuntu-22.04
基本使用
从 Windows Terminal 打开 WSL wsl
访问 Windows 文件 ls /mnt/c/Users/YourName/Desktop
cd /mnt/c/
从 Windows 访问 WSL 文件 在资源管理器地址栏输入：\\wsl$\Ubuntu
在 WSL 中安装工具 sudo apt update && sudo apt upgrade -y
sudo apt install -y python3 python3-pip git curl wget
WSL2 vs 虚拟机
对比维度WSL2传统虚拟机 (VMware / VirtualBox)
启动速度秒级启动分钟级启动
资源占用轻量，按需分配内存固定分配 CPU / 内存
文件互通Windows ↔ Linux 双向访问需要共享文件夹配置
Docker 集成Docker Desktop 原生支持 WSL2 后端需在虚拟机内单独安装
VS Code 集成Remote-WSL 插件无缝连接需要 SSH 连接
网络与 Windows 共享网络NAT / 桥接配置复杂

Python 自动检测脚本（保存为 check_env.py 运行）
check_env.py — 环境验证脚本 import subprocess
import sys
import shutil

PASS = "\033[92m[PASS]\033[0m"
FAIL = "\033[91m[FAIL]\033[0m"

def check_cmd(name, cmd):
    try:
        result = subprocess.run(cmd, capture_output=True, text=True, timeout=15)
        version = result.stdout.strip() or result.stderr.strip()
        first_line = version.split("\n")[0]
        print(f" {PASS} {name}: {first_line}")
        return True
    except Exception:
        print(f" {FAIL} {name}: 未安装或不可用")
        return False

def check_python_pkg(name, import_name=None):
    pkg = import_name or name
    try:
        mod = __import__(pkg)
        ver = getattr(mod, "__version__", "installed")
        print(f" {PASS} {name}: {ver}")
        return True
    except ImportError:
        print(f" {FAIL} {name}: 未安装")
        return False

print("\n===== Windows 大数据环境检测 =====\n")

total = 0
passed = 0

checks = [
    ("Python", ["python", "--version"]),
    ("pip", ["pip", "--version"]),
    ("Java", ["java", "-version"]),
    ("Git", ["git", "--version"]),
    ("Docker", ["docker", "--version"]),
]

for name, cmd in checks:
    total += 1
    if check_cmd(name, cmd):
        passed += 1

pkgs = [("PySpark", "pyspark"), ("DuckDB", "duckdb"), ("pandas", None), ("numpy", None)]

for name, imp in pkgs:
    total += 1
    if check_python_pkg(name, imp):
        passed += 1

print(f"\n===== 结果：{passed}/{total} 通过 =====\n")

if passed == total:
    print("所有工具安装完成，可以开始大数据学习了！")
else:
    print("部分工具未安装，请根据上方提示补全。")
