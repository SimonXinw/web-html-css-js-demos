四、基础 SQL 练习 5 题
Q1：各城市订单数和 GMV 基础
统计每个城市的已支付订单数和GMV（总金额），按 GMV 降序排列。
考察点 GROUP BY + 聚合函数 + WHERE 过滤 + ORDER BY
查看答案
SELECT u.city, COUNT(*) AS order_cnt, SUM(o.total_amount) AS gmv FROM orders o JOIN users u ON o.user_id = u.user_id WHERE o.status = 'paid' GROUP BY u.city ORDER BY gmv DESC;
Q2：2024年4月已支付订单 基础
查询 2024年4月 所有已支付订单的 order_id、用户名、商品名、金额，按日期升序。
考察点 日期范围过滤 + 多表 JOIN
查看答案
SELECT o.order_id, u.username, p.product_name, o.total_amount, o.order_date FROM orders o JOIN users u ON o.user_id = u.user_id JOIN products p ON o.product_id = p.product_id WHERE o.status = 'paid' AND o.order_date >= '2024-04-01' AND o.order_date < '2024-05-01' ORDER BY o.order_date;
Q3：客单价 Top 10 订单 基础
查询单笔金额最高的前 10 笔已支付订单，显示用户名、商品名、金额。
考察点 ORDER BY + LIMIT
查看答案
SELECT o.order_id, u.username, p.product_name, o.total_amount FROM orders o JOIN users u ON o.user_id = u.user_id JOIN products p ON o.product_id = p.product_id WHERE o.status = 'paid' ORDER BY o.total_amount DESC LIMIT 10;
Q4：各品类订单数和销售额 基础
按商品品类统计已支付订单数和总销售额，找出最畅销的品类。
考察点 JOIN + GROUP BY 维度列
查看答案
SELECT p.category, COUNT(*) AS order_cnt, SUM(o.total_amount) AS total_sales FROM orders o JOIN products p ON o.product_id = p.product_id WHERE o.status = 'paid' GROUP BY p.category ORDER BY total_sales DESC;
Q5：未下单用户列表 基础
找出注册但从未下过已支付订单的用户（user_id、用户名、国家）。
考察点 LEFT JOIN + IS NULL / NOT IN / NOT EXISTS
查看答案
写法一：LEFT JOIN + IS NULL SELECT u.user_id, u.username, u.country FROM users u LEFT JOIN orders o ON u.user_id = o.user_id AND o.status = 'paid' WHERE o.order_id IS NULL;
写法二：NOT EXISTS SELECT u.user_id, u.username, u.country FROM users u WHERE NOT EXISTS ( SELECT 1 FROM orders o WHERE o.user_id = u.user_id AND o.status = 'paid' );

五、JOIN 进阶 4 题
Q6：三表 JOIN — 订单明细宽表 进阶
构建一张订单明细宽表，包含：order_id、用户名、国家、商品名、品类、品牌、金额、日期。只保留已支付订单。
考察点 三表 JOIN + 字段选择，模拟 DWD 层宽表构建
查看答案
SELECT o.order_id, u.username, u.country, p.product_name, p.category, p.brand, o.total_amount, o.order_date FROM orders o JOIN users u ON o.user_id = u.user_id JOIN products p ON o.product_id = p.product_id WHERE o.status = 'paid' ORDER BY o.order_date;
Q7：同天购买多品类的用户 进阶
找出在同一天购买了2个及以上不同品类商品的用户。
考察点 JOIN + GROUP BY 多列 + HAVING + COUNT(DISTINCT)
查看答案
SELECT u.username, o.order_date, COUNT(DISTINCT p.category) AS cat_cnt FROM orders o JOIN users u ON o.user_id = u.user_id JOIN products p ON o.product_id = p.product_id WHERE o.status = 'paid' GROUP BY u.username, o.order_date HAVING COUNT(DISTINCT p.category) >= 2 ORDER BY o.order_date;
Q8：每位用户首购品类分布 进阶
统计每位用户首次购买的商品品类，然后汇总各品类的首购人数。
考察点 子查询/CTE + MIN(order_date) + JOIN
查看答案
WITH first_order AS ( SELECT user_id, MIN(order_date) AS first_date FROM orders WHERE status = 'paid' GROUP BY user_id ) SELECT p.category, COUNT(*) AS first_purchase_users FROM first_order fo JOIN orders o ON fo.user_id = o.user_id AND fo.first_date = o.order_date AND o.status = 'paid' JOIN products p ON o.product_id = p.product_id GROUP BY p.category ORDER BY first_purchase_users DESC;
Q9：用户最常购买的品牌 进阶
找出每位用户下单次数最多的品牌（如有并列取任一）。
考察点 窗口函数 ROW_NUMBER + GROUP BY + 子查询
查看答案
WITH brand_cnt AS ( SELECT o.user_id, p.brand, COUNT(*) AS buy_cnt FROM orders o JOIN products p ON o.product_id = p.product_id WHERE o.status = 'paid' GROUP BY o.user_id, p.brand ), ranked AS ( SELECT *, ROW_NUMBER() OVER (PARTITION BY user_id ORDER BY buy_cnt DESC) AS rn FROM brand_cnt ) SELECT u.username, r.brand, r.buy_cnt FROM ranked r JOIN users u ON r.user_id = u.user_id WHERE r.rn = 1 ORDER BY r.buy_cnt DESC;

六、窗口函数实战 7 题
Q10：ROW_NUMBER — 每用户最近一单 高频
取每位用户最近一笔已支付订单的全部信息。
考察点 ROW_NUMBER() OVER (PARTITION BY ... ORDER BY ... DESC)
查看答案
SELECT * FROM ( SELECT o.*, ROW_NUMBER() OVER (PARTITION BY o.user_id ORDER BY o.order_date DESC) AS rn FROM orders o WHERE o.status = 'paid' ) sub WHERE rn = 1;
Q11：RANK — 品类内销售额排名 高频
在每个品类内，按商品总销售额排名，找出每个品类的 Top 2。
考察点 RANK() vs DENSE_RANK() + PARTITION BY category
查看答案
WITH product_sales AS ( SELECT p.category, p.product_name, SUM(o.total_amount) AS total_sales FROM orders o JOIN products p ON o.product_id = p.product_id WHERE o.status = 'paid' GROUP BY p.category, p.product_name ), ranked AS ( SELECT *, RANK() OVER (PARTITION BY category ORDER BY total_sales DESC) AS rk FROM product_sales ) SELECT * FROM ranked WHERE rk <= 2;
Q12：LAG — 每用户订单间隔天数 进阶
计算每位用户相邻两笔订单之间的间隔天数。
考察点 LAG() + DATE_DIFF / date subtraction
查看答案
SELECT user_id, order_id, order_date, LAG(order_date) OVER (PARTITION BY user_id ORDER BY order_date) AS prev_date, order_date - LAG(order_date) OVER (PARTITION BY user_id ORDER BY order_date) AS gap_days FROM orders WHERE status = 'paid' ORDER BY user_id, order_date;
Q13：SUM OVER — 用户累计消费 高频
计算每位用户按时间顺序的累计消费金额。
考察点 SUM() OVER (PARTITION BY ... ORDER BY ... ROWS UNBOUNDED PRECEDING)
查看答案
SELECT user_id, order_id, order_date, total_amount, SUM(total_amount) OVER ( PARTITION BY user_id ORDER BY order_date ROWS BETWEEN UNBOUNDED PRECEDING AND CURRENT ROW ) AS cumulative_amount FROM orders WHERE status = 'paid' ORDER BY user_id, order_date;
Q14：NTILE — 用户消费分层 进阶
将所有用户按总消费金额分为高/中/低三个等级（NTILE(3)）。
考察点 NTILE() + 子查询聚合
查看答案
WITH user_total AS ( SELECT user_id, SUM(total_amount) AS total_spend FROM orders WHERE status = 'paid' GROUP BY user_id ) SELECT u.username, ut.total_spend, NTILE(3) OVER (ORDER BY ut.total_spend DESC) AS spend_tier, CASE NTILE(3) OVER (ORDER BY ut.total_spend DESC) WHEN 1 THEN '高消费' WHEN 2 THEN '中消费' WHEN 3 THEN '低消费' END AS tier_label FROM user_total ut JOIN users u ON ut.user_id = u.user_id ORDER BY ut.total_spend DESC;
Q15：FIRST_VALUE — 用户首购商品 进阶
查询每位用户的每笔订单，同时显示该用户首次购买的商品名。
考察点 FIRST_VALUE() OVER (PARTITION BY ... ORDER BY ...)
查看答案
SELECT o.user_id, u.username, o.order_id, o.order_date, p.product_name, FIRST_VALUE(p.product_name) OVER ( PARTITION BY o.user_id ORDER BY o.order_date ) AS first_product FROM orders o JOIN users u ON o.user_id = u.user_id JOIN products p ON o.product_id = p.product_id WHERE o.status = 'paid' ORDER BY o.user_id, o.order_date;
Q16：滑动平均 — 每日 GMV 的 3 日移动平均 进阶
计算每日 GMV，然后求最近 3 天的移动平均 GMV。
考察点 AVG() OVER (ORDER BY ... ROWS BETWEEN 2 PRECEDING AND CURRENT ROW)
查看答案
WITH daily_gmv AS ( SELECT order_date, SUM(total_amount) AS gmv FROM orders WHERE status = 'paid' GROUP BY order_date ) SELECT order_date, gmv, ROUND(AVG(gmv) OVER ( ORDER BY order_date ROWS BETWEEN 2 PRECEDING AND CURRENT ROW ), 2) AS ma3_gmv FROM daily_gmv ORDER BY order_date;

七、CTE 与复杂查询 5 题
Q17：RFM 用户分层 高频
基于 RFM 模型为每位用户打分：R（最近购买距今天数）、F（购买频次）、M（累计消费金额），并用 NTILE(3) 分别打 1-3 分后求总分。
考察点 多层 CTE + 聚合 + NTILE 窗口函数
查看答案
WITH rfm_raw AS ( SELECT user_id, CURRENT_DATE - MAX(order_date) AS recency_days, COUNT(*) AS frequency, SUM(total_amount) AS monetary FROM orders WHERE status = 'paid' GROUP BY user_id ), rfm_scored AS ( SELECT *, NTILE(3) OVER (ORDER BY recency_days ASC) AS r_score, NTILE(3) OVER (ORDER BY frequency DESC) AS f_score, NTILE(3) OVER (ORDER BY monetary DESC) AS m_score FROM rfm_raw ) SELECT u.username, u.country, rs.recency_days, rs.frequency, ROUND(rs.monetary, 2) AS monetary, rs.r_score, rs.f_score, rs.m_score, rs.r_score + rs.f_score + rs.m_score AS rfm_total, CASE WHEN rs.r_score + rs.f_score + rs.m_score >= 8 THEN '高价值用户' WHEN rs.r_score + rs.f_score + rs.m_score >= 5 THEN '中价值用户' ELSE '低价值用户' END AS user_tier FROM rfm_scored rs JOIN users u ON rs.user_id = u.user_id ORDER BY rfm_total DESC;
Q18：日期序列填充 — 无订单日补零 进阶
生成 2024-03-01 到 2024-05-31 的完整日期序列，LEFT JOIN 订单表，没有订单的日期 GMV 显示为 0。
考察点 generate_series + LEFT JOIN + COALESCE
查看答案
WITH date_series AS ( SELECT CAST(dt AS DATE) AS dt FROM generate_series(DATE '2024-03-01', DATE '2024-05-31', INTERVAL 1 DAY) AS t(dt) ), daily_gmv AS ( SELECT order_date, SUM(total_amount) AS gmv FROM orders WHERE status = 'paid' GROUP BY order_date ) SELECT ds.dt, COALESCE(dg.gmv, 0) AS gmv FROM date_series ds LEFT JOIN daily_gmv dg ON ds.dt = dg.order_date ORDER BY ds.dt;
Q19：漏斗分析 — 页面转化率 高频
基于 page_views 表，统计 /home → /products → /cart → /checkout 各步骤的独立用户数和转化率。
考察点 CTE + CASE WHEN + COUNT(DISTINCT) + 漏斗思维
查看答案
WITH funnel AS ( SELECT COUNT(DISTINCT CASE WHEN page_url = '/home' THEN user_id END) AS step1_home, COUNT(DISTINCT CASE WHEN page_url LIKE '/products%' THEN user_id END) AS step2_product, COUNT(DISTINCT CASE WHEN page_url = '/cart' THEN user_id END) AS step3_cart, COUNT(DISTINCT CASE WHEN page_url = '/checkout' THEN user_id END) AS step4_checkout FROM page_views ) SELECT step1_home, step2_product, step3_cart, step4_checkout, ROUND(step2_product * 100.0 / step1_home, 1) AS home_to_product_pct, ROUND(step3_cart * 100.0 / step2_product, 1) AS product_to_cart_pct, ROUND(step4_checkout * 100.0 / step3_cart, 1) AS cart_to_checkout_pct, ROUND(step4_checkout * 100.0 / step1_home, 1) AS overall_cvr_pct FROM funnel;
Q20：月度留存率 进阶
计算3月购买用户在 4月、5月的留存率（后续月份仍有购买的用户占比）。
考察点 多 CTE + CROSS JOIN 月份 + LEFT JOIN + 留存定义
查看答案
WITH mar_users AS ( SELECT DISTINCT user_id FROM orders WHERE status = 'paid' AND order_date >= '2024-03-01' AND order_date < '2024-04-01' ), user_months AS ( SELECT DISTINCT user_id, DATE_TRUNC('month', order_date) AS order_month FROM orders WHERE status = 'paid' ) SELECT um.order_month, COUNT(DISTINCT um.user_id) AS retained_users, (SELECT COUNT(*) FROM mar_users) AS base_users, ROUND(COUNT(DISTINCT um.user_id) * 100.0 / (SELECT COUNT(*) FROM mar_users), 1) AS retention_pct FROM mar_users mu JOIN user_months um ON mu.user_id = um.user_id WHERE um.order_month >= '2024-03-01' GROUP BY um.order_month ORDER BY um.order_month;
Q21：月度 GMV 同比环比 进阶
计算每月 GMV，以及相对上月的环比增长率。
考察点 DATE_TRUNC + LAG + 百分比计算
查看答案
WITH monthly AS ( SELECT DATE_TRUNC('month', order_date) AS month, SUM(total_amount) AS gmv FROM orders WHERE status = 'paid' GROUP BY DATE_TRUNC('month', order_date) ) SELECT month, ROUND(gmv, 2) AS gmv, ROUND(LAG(gmv) OVER (ORDER BY month), 2) AS prev_gmv, ROUND((gmv - LAG(gmv) OVER (ORDER BY month)) / LAG(gmv) OVER (ORDER BY month) * 100, 1) AS mom_growth_pct FROM monthly ORDER BY month;

八、DuckDB 高级特性 DuckDB 独有语法
QUALIFY — 窗口函数直接筛选 DuckDB特色
QUALIFY 子句在窗口函数计算之后过滤，取代外层嵌套子查询，极大简化代码。
传统写法 vs QUALIFY 写法 -- 传统：需要嵌套子查询 SELECT * FROM ( SELECT *, ROW_NUMBER() OVER (PARTITION BY user_id ORDER BY order_date DESC) AS rn FROM orders WHERE status = 'paid' ) WHERE rn = 1; -- DuckDB QUALIFY：一步搞定 SELECT *, ROW_NUMBER() OVER (PARTITION BY user_id ORDER BY order_date DESC) AS rn FROM orders WHERE status = 'paid' QUALIFY rn = 1;
更多示例：每品类销售额 Top 1 的商品 SELECT p.category, p.product_name, SUM(o.total_amount) AS sales, RANK() OVER (PARTITION BY p.category ORDER BY SUM(o.total_amount) DESC) AS rk FROM orders o JOIN products p ON o.product_id = p.product_id WHERE o.status = 'paid' GROUP BY p.category, p.product_name QUALIFY rk = 1;
PIVOT / UNPIVOT — 行列转换 进阶
PIVOT：按月份将 GMV 展开为列 PIVOT ( SELECT DATE_TRUNC('month', order_date) AS month, total_amount FROM orders WHERE status = 'paid' ) ON month USING SUM(total_amount);
UNPIVOT：将列转行 -- 假设有宽表 monthly_sales(jan, feb, mar) -- UNPIVOT 可将月份列转为行 CREATE TABLE wide_sales AS SELECT 'US' AS country, 1000 AS jan, 1200 AS feb, 1500 AS mar; UNPIVOT wide_sales ON jan, feb, mar INTO NAME month VALUE sales;
EXCLUDE / REPLACE — 简化 SELECT 进阶
EXCLUDE：选所有列但排除指定列 SELECT * EXCLUDE (status) FROM orders WHERE status = 'paid';
REPLACE：选所有列但替换指定列的表达式 SELECT * REPLACE (ROUND(total_amount, 0) AS total_amount) FROM orders LIMIT 5;
这两个语法在列很多的宽表场景尤其好用，避免逐一列出字段名。
read_csv_auto / COPY TO — 文件读写 实用
直接查询 CSV 文件 SELECT * FROM read_csv_auto('orders.csv') LIMIT 10; -- 指定分隔符、编码 SELECT * FROM read_csv('data.csv', delim=',', header=true, encoding='utf-8' );
导出为 Parquet 文件 COPY (SELECT * FROM orders WHERE status = 'paid') TO 'paid_orders.parquet' (FORMAT PARQUET); -- 再读回来验证 SELECT COUNT(*) FROM read_parquet('paid_orders.parquet');
直接对 Parquet 文件做分析 SELECT order_date, SUM(total_amount) AS gmv FROM read_parquet('paid_orders.parquet') GROUP BY order_date ORDER BY order_date;
