一、Excel核心函数（VLOOKUP / SUMIF / IF） 必背
VLOOKUP 查找匹配 必背
根据查找值在表格首列中查找，返回对应行指定列的值。
语法 =VLOOKUP(查找值, 数据范围, 列序号, 精确/模糊)
  • 查找值：要匹配的值（如商品ID）
  • 数据范围：查找列必须在首列
  • 列序号：返回第几列
  • FALSE：精确匹配；TRUE：近似匹配
示例：根据商品ID查价格 =VLOOKUP(A2, 商品表!$A:$D, 3, FALSE)
SUMIF 条件求和
对满足条件的单元格求和，是日报/周报/月报的核心函数。
语法 =SUMIF(条件区域, 条件, 求和区域)
  • 条件区域：判断条件的列
  • 条件：如 ">100"、">=2024-01-01"、"*北京*"
  • 求和区域：可选，省略时对条件区域求和
示例：某品类销售额 =SUMIF(B:B, "电子产品", C:C)
IF 条件判断
根据条件返回不同值，可嵌套实现复杂逻辑。
语法 =IF(条件, 真时返回值, 假时返回值)
  • 嵌套IF：多条件分级（如业绩等级划分）
  • IFERROR：VLOOKUP 失败时返回默认值
示例：VLOOKUP 容错 =IFERROR(VLOOKUP(A2, 表!$A:$D, 3, FALSE), "未找到")
常用组合函数
函数用途
INDEX+MATCH替代 VLOOKUP，支持反向查找
SUMIFS多条件求和（SUMIF 升级版）
COUNTIF条件计数
AVERAGEIF条件求平均

二、数据透视表 多维分析利器
数据透视表核心概念 核心
将明细数据按维度聚合，支持拖拽式分析，无需写公式。
原始明细表 → 插入 → 数据透视表 → 拖拽行列/值 → 即时汇总
  • 行：分组维度（如品类、地区、日期）
  • 列：交叉维度（可选）
  • 值：聚合方式（求和、计数、平均）
  • 筛选器：全局过滤
透视表 vs 公式
维度数据透视表公式
灵活性拖拽变更维度需改公式
性能大数据量易卡顿公式计算可控
适用场景探索性分析固定报表
透视表高级技巧
  • 计算字段：在透视表内新增字段（如利润率 = 利润/销售额）
  • 分组：日期按年/月/季度分组；数值按区间分组
  • 切片器：可视化筛选器，多透视表联动
  • 数据源：使用动态区域或 Power Query 保证数据源可扩展

三、条件格式 数据高亮与预警
条件格式类型
类型用途
数据条条形图直观显示数值大小
色阶颜色渐变表示高低
图标集箭头/红绿灯表示趋势
突出显示大于/小于/等于某值
自定义公式复杂条件（如隔行着色）
条件格式应用场景
  • KPI 预警：低于目标标红，高于目标标绿
  • 同比环比：正增长用绿色箭头，负增长用红色
  • 数据条：销售额/占比直观对比
  • 重复值：快速发现重复数据
示例：销售额大于阈值标绿 条件格式 → 新建规则 → 使用公式
=C2>=10000 → 格式：绿色填充

四、图表类型（折线/柱状/饼/散点/漏斗） 选对图表类型
图表类型对比
图表适用场景注意
折线图时间趋势、连续变化X 轴需有序
柱状图分类对比、排名类别不宜过多
饼图占比、构成类别≤5，避免过多
散点图相关性、分布两变量关系
漏斗图转化漏斗、流失分析层级递减
图表选择流程
看趋势？→ 折线图
看对比？→ 柱状图
看占比？→ 饼图
看关系？→ 散点图
看转化？→ 漏斗图
电商常用：GMV 趋势→折线；品类对比→柱状；渠道占比→饼；用户漏斗→漏斗图

五、图表设计原则 让数据说话
图表设计原则 必背
  • 少即是多：去掉无关元素（网格线、3D 效果、装饰）
  • 数据墨水比：数据元素占比应高，非数据元素占比应低
  • 颜色语义：红=负/降，绿=正/升；保持一致性
  • 标题即结论：标题用结论描述，而非"销售额趋势"
  • 避免误导：Y 轴不截断、不随意缩放
常见设计误区
误区正确做法
3D 饼图用 2D 饼图或柱状图
Y 轴从非 0 开始从 0 开始，避免夸大差异
过多颜色主色+辅助色，不超过 5 种
无数据标签关键数据加标签

六、Dashboard 设计 一屏看全
Dashboard 设计原则
核心指标 → 次要指标 → 明细/钻取
  • F 型布局：左上放核心 KPI，符合阅读习惯
  • 层次清晰：大数字→小图表→明细表
  • 联动筛选：筛选器影响所有图表
  • 刷新频率：标注数据更新时间
Dashboard 典型结构
电商运营 Dashboard 示例 第1行：GMV | UV | 订单数 | 转化率 | 客单价
第2行：GMV 趋势折线图（7日/30日）
第3行：品类柱状图 | 渠道饼图 | 地域热力图
第4行：Top10 商品表 | 异常订单表

七、BI 工具（Tableau / PowerBI / Superset） 企业级可视化
BI 工具对比
工具特点适用场景
Tableau拖拽式、交互强、学习曲线陡专业分析、跨境公司常用
Power BI微软生态、DAX 公式、与 Office 集成企业内部分析
Superset开源、支持多种数据源、可自部署技术团队、成本敏感
MetabaseSQL 友好、配置简单非技术人员自助
BI 工具核心能力
  • 数据连接：多数据源、实时/定时刷新
  • 可视化：丰富图表类型、自定义主题
  • 交互：筛选、钻取、联动、下钻
  • 分享：权限控制、定时邮件、嵌入
Excel 适合小数据、一次性分析；BI 适合大数据、持续监控、自助分析

八、电商报表（日报/周报/月报） 业务实战
报表层级与粒度
报表粒度核心指标
日报按天GMV、UV、订单数、转化率、异常
周报按周环比、同比、品类/渠道趋势
月报按月目标达成、复盘、下月规划
电商报表设计流程
明确受众 → 确定指标 → 数据口径 → 模板设计 → 自动化
  • 日报：运营看异常、及时调整
  • 周报：管理层看趋势、做决策
  • 月报：复盘+规划，含同比环比
报表自动化思路
  • 数据源：从数仓/BI 定时同步，或 API 拉取
  • 模板：固定格式，只更新数据区域
  • 定时任务：Airflow / 调度系统每日跑数
  • 分发：邮件/钉钉/企业微信自动推送
⚠️
九、可视化避坑 常见错误
可视化避坑清单 必背
  • Y 轴截断：从非 0 开始会夸大差异，误导读者
  • 双 Y 轴：两轴尺度不同易混淆，慎用
  • 饼图过多：类别超过 5 个用柱状图
  • 颜色滥用：无语义的彩虹色，红绿混用
  • 3D 图表：遮挡数据、难以阅读
  • 数据口径不一致：同一指标不同定义
数据展示陷阱
陷阱正确做法
幸存者偏差展示全量样本或说明筛选条件
辛普森悖论拆分维度验证，避免整体掩盖分组
混淆相关与因果只展示相关性，不轻易下因果结论
忽略基准同比环比需有基准期

实战 Dashboard 设计流程 核心

Dashboard 检查清单
  • 核心指标是否在首屏？
  • 图表类型是否合适？
