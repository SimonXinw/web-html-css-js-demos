一、Docker 基础概念速览 Windows + Docker Desktop
镜像 / 容器 / 卷 必知
  • 镜像（Image）：只读模板，包含运行环境和应用程序，类似"安装包"
  • 容器（Container）：镜像的运行实例，可启动/停止/删除，类似"正在运行的程序"
  • 卷（Volume）：持久化存储，容器删除后数据仍然保留
  • Dockerfile：用代码描述镜像构建过程的文本文件
  • Registry：镜像仓库，如 Docker Hub、阿里云镜像服务
类比理解镜像 = 类（Class） 容器 = 实例（Instance） 卷 = 外接硬盘（不随容器销毁）
Docker vs 虚拟机
维度Docker 容器虚拟机
启动速度秒级分钟级
资源占用MB 级（共享宿主内核）GB 级（完整 OS）
隔离级别进程级隔离硬件级隔离
镜像大小几十 MB ～ 几百 MB几 GB ～ 十几 GB
性能损耗几乎无损耗有虚拟化开销
适用场景微服务、开发测试、CI/CD需要完整 OS 的场景
docker-compose 多容器编排 核心
用一个 YAML 文件定义多个服务（MySQL + Kafka + ClickHouse），一条命令全部启动。

Step 1 — MySQL 读取 → Kafka 发送 Python
安装依赖pip install pymysql kafka-python clickhouse-driver
mysql_to_kafka.pyimport json import pymysql from kafka import KafkaProducer producer = KafkaProducer( bootstrap_servers='localhost:9092', value_serializer=lambda v: json.dumps(v, default=str).encode('utf-8') ) conn = pymysql.connect( host='127.0.0.1', port=3306, user='root', password='root123', database='ecommerce', cursorclass=pymysql.cursors.DictCursor ) with conn.cursor() as cursor: cursor.execute("SELECT * FROM orders") rows = cursor.fetchall() for row in rows: producer.send('order-events', value=row) print(f"Sent: order_id={row['order_id']}, amount={row['amount']}") producer.flush() producer.close() conn.close() print(f"\nDone! Sent {len(rows)} orders to Kafka topic 'order-events'")
Step 2 — Kafka 消费 → ClickHouse 写入 Python
kafka_to_clickhouse.pyimport json from kafka import KafkaConsumer from clickhouse_driver import Client consumer = KafkaConsumer( 'order-events', bootstrap_servers='localhost:9092', auto_offset_reset='earliest', group_id='ck-writer', value_deserializer=lambda m: json.loads(m.decode('utf-8')), consumer_timeout_ms=10000 ) ck = Client(host='127.0.0.1', port=9000, user='default', password='ck123') ck.execute("CREATE DATABASE IF NOT EXISTS ecommerce") ck.execute(""" CREATE TABLE IF NOT EXISTS ecommerce.orders ( order_id UInt32, user_id UInt32, product_id UInt32, quantity UInt16, amount Decimal(10,2), status String, order_date Date, order_time DateTime ) ENGINE = MergeTree() PARTITION BY toYYYYMM(order_date) ORDER BY (order_date, order_id) """) count = 0 for msg in consumer: row = msg.value ck.execute( "INSERT INTO ecommerce.orders VALUES", [(row['order_id'], row['user_id'], row['product_id'], row.get('quantity', 1), float(row['amount']), row.get('status', 'unknown'), str(row['order_time'])[:10], str(row['order_time']))] ) count += 1 print(f"Written: order_id={row['order_id']}") consumer.close() print(f"\nDone! Written {count} orders to ClickHouse")

执行完整操作流程# 1. 启动所有服务 docker-compose up -d # 2. 等待服务就绪（约 30 秒） docker-compose ps # 3. 创建 Kafka Topic docker exec pipeline-kafka kafka-topics --create ^ --bootstrap-server localhost:9092 ^ --replication-factor 1 --partitions 3 ^ --topic order-events # 4. 运行 MySQL → Kafka 脚本 python mysql_to_kafka.py # 5. 运行 Kafka → ClickHouse 脚本 python kafka_to_clickhouse.py # 6. 在 ClickHouse 中查询验证 docker exec -it pipeline-clickhouse clickhouse-client --password ck123 ^ -q "SELECT order_date, count() AS cnt, sum(amount) AS gmv FROM ecommerce.orders GROUP BY order_date ORDER BY order_date"

创建 DAG — 每日订单统计 Python
dags/daily_order_report.py（放在 dags 目录下）from datetime import datetime, timedelta from airflow import DAG from airflow.operators.python import PythonOperator from airflow.operators.bash import BashOperator default_args = { 'owner': 'data-team', 'depends_on_past': False, 'email_on_failure': False, 'retries': 1, 'retry_delay': timedelta(minutes=5), } def extract_orders(**context): """模拟从 MySQL 抽取订单数据""" ds = context['ds'] print(f"Extracting orders for date: {ds}") print("Found 156 new orders") return 156 def transform_data(**context): """模拟数据清洗和转换""" ti = context['ti'] order_count = ti.xcom_pull(task_ids='extract_orders') print(f"Transforming {order_count} orders...") print("Cleaned: removed 3 duplicates, fixed 2 null values") return order_count - 3 def load_to_warehouse(**context): """模拟加载到数仓""" ti = context['ti'] clean_count = ti.xcom_pull(task_ids='transform_data') print(f"Loading {clean_count} clean orders to ClickHouse...") print("Load complete!") with DAG( dag_id='daily_order_report', default_args=default_args, description='每日订单数据 ETL 任务', schedule_interval='0 2 * * *', start_date=datetime(2025, 3, 1), catchup=False, tags=['etl', 'orders'], ) as dag: t1 = PythonOperator( task_id='extract_orders', python_callable=extract_orders, ) t2 = PythonOperator( task_id='transform_data', python_callable=transform_data, ) t3 = PythonOperator( task_id='load_to_warehouse', python_callable=load_to_warehouse, ) t4 = BashOperator( task_id='notify_completion', bash_command='echo "Daily order report ETL completed at $(date)"', ) t1 >> t2 >> t3 >> t4

Windows 上 Airflow 的坑：
  • 路径挂载：Windows 路径用正斜杠 ./dags，不要用 .\dags
  • 文件权限：Docker Desktop 默认共享 C 盘，如在其他盘需在 Settings → Resources → File Sharing 添加
  • 行尾符：DAG Python 文件确保使用 LF（不是 CRLF），否则可能报语法错误
  • 内存：Airflow 至少需要 4GB 内存，Docker Desktop 默认分配可能不足

Python 连接 Redis 示例 Python
安装依赖pip install redis
redis_realtime_gmv.pyimport redis import random from datetime import datetime r = redis.Redis(host='127.0.0.1', port=6379, decode_responses=True) today = datetime.now().strftime('%Y-%m-%d') gmv_key = f"realtime_gmv:{today}" count_key = f"order_count:{today}" r.set(gmv_key, 0) r.set(count_key, 0) print(f"Simulating real-time orders for {today}...\n") for i in range(20): amount = round(random.uniform(50, 2000), 2) amount_cents = int(amount * 100) r.incrby(gmv_key, amount_cents) r.incr(count_key) r.hset(f"latest_order:{today}", mapping={ 'order_id': f"ORD-{1000+i}", 'amount': str(amount), 'timestamp': datetime.now().isoformat() }) current_gmv = int(r.get(gmv_key)) / 100 current_count = r.get(count_key) print(f" Order #{i+1}: ¥{amount:.2f} | " f"GMV: ¥{current_gmv:,.2f} | " f"Orders: {current_count}") print(f"\n--- Final Stats ---") print(f"Total GMV: ¥{int(r.get(gmv_key))/100:,.2f}") print(f"Total Orders: {r.get(count_key)}") print(f"Latest Order: {r.hgetall(f'latest_order:{today}')}")
大数据场景中 Redis 的典型用途：
  • 实时指标缓存：累计 GMV、在线人数、PV/UV 实时统计
  • 排行榜：Sorted Set 实现商品销量排行（ZADD / ZREVRANGE）
  • 去重计数：HyperLogLog 实现 UV 统计（误差 < 1%，内存极小）
  • 分布式锁：防止重复消费、幂等处理
  • 维度缓存：Flink 实时 JOIN 维度表时，用 Redis 缓存维度数据加速

docker-compose.yml（MySQL + Kafka + ClickHouse + Redis）version: '3.8' services: # ===== MySQL ===== mysql: image: mysql:8.0 container_name: bigdata-mysql restart: always ports: - "3306:3306" environment: MYSQL_ROOT_PASSWORD: root123 MYSQL_DATABASE: ecommerce TZ: Asia/Shanghai volumes: - mysql_data:/var/lib/mysql - ./init.sql:/docker-entrypoint-initdb.d/init.sql command: --character-set-server=utf8mb4 --collation-server=utf8mb4_unicode_ci healthcheck: test: ["CMD", "mysqladmin", "ping", "-h", "localhost"] interval: 10s retries: 5 # ===== Zookeeper ===== zookeeper: image: confluentinc/cp-zookeeper:7.5.0 container_name: bigdata-zookeeper restart: always environment: ZOOKEEPER_CLIENT_PORT: 2181 ZOOKEEPER_TICK_TIME: 2000 # ===== Kafka ===== kafka: image: confluentinc/cp-kafka:7.5.0 container_name: bigdata-kafka restart: always depends_on: - zookeeper ports: - "9092:9092" environment: KAFKA_BROKER_ID: 1 KAFKA_ZOOKEEPER_CONNECT: zookeeper:2181 KAFKA_ADVERTISED_LISTENERS: PLAINTEXT://localhost:9092 KAFKA_OFFSETS_TOPIC_REPLICATION_FACTOR: 1 KAFKA_AUTO_CREATE_TOPICS_ENABLE: "true" KAFKA_LOG_RETENTION_HOURS: 168 # ===== ClickHouse ===== clickhouse: image: clickhouse/clickhouse-server:23.8 container_name: bigdata-clickhouse restart: always ports: - "8123:8123" - "9000:9000" volumes: - ch_data:/var/lib/clickhouse environment: CLICKHOUSE_USER: default CLICKHOUSE_PASSWORD: ck123 ulimits: nofile: soft: 262144 hard: 262144 # ===== Redis ===== redis: image: redis:7-alpine container_name: bigdata-redis restart: always ports: - "6379:6379" volumes: - redis_data:/data command: redis-server --appendonly yes --maxmemory 512mb --maxmemory-policy allkeys-lru volumes: mysql_data: ch_data: redis_data:

启动 / 停止 / 管理命令
一键启动全套# 后台启动所有服务 docker-compose up -d # 查看所有服务状态 docker-compose ps # 查看各服务日志 docker-compose logs -f mysql docker-compose logs -f kafka docker-compose logs -f clickhouse docker-compose logs -f redis
停止与重启# 停止所有服务（保留数据卷） docker-compose stop # 停止并删除容器（保留数据卷） docker-compose down # 停止并删除容器 + 数据卷（彻底清理） docker-compose down -v # 重启所有服务 docker-compose restart # 只重启某个服务 docker-compose restart clickhouse

九、常见问题排查 Windows 环境 FAQ
端口占用
检查端口占用# 查看 3306 端口被哪个进程占用 netstat -ano | findstr :3306 # 通过 PID 查看进程名称 tasklist | findstr <PID> # 强制结束占用进程（谨慎使用） taskkill /PID <PID> /F # 或修改 docker-compose.yml 映射到其他端口 ports: - "3307:3306" # 宿主机用 3307
容器无法启动
排查步骤# 1. 查看容器状态 docker-compose ps # 2. 查看错误日志（关键！） docker logs <容器名> docker logs --tail 50 <容器名> # 3. 常见错误及解决 # "port is already allocated" → 端口冲突，改端口 # "no space left on device" → 磁盘满，清理空间 # "OOM killed" → 内存不足，增大 Docker 内存分配
磁盘空间不足
清理 Docker 占用空间# 查看 Docker 磁盘占用 docker system df # 清理已停止的容器 + 无用镜像 + 无用网络 docker system prune -f # 清理所有未使用的镜像（包括中间层） docker image prune -a -f # 清理无用数据卷（谨慎！会删数据） docker volume prune -f # 查看各镜像大小 docker images --format "table {{.Repository}}\t{{.Tag}}\t{{.Size}}"
Windows 特有问题
  • 以管理员运行：部分操作需要管理员权限的 PowerShell / CMD
  • WSL2 后端：Docker Desktop 推荐使用 WSL2 后端，性能更好。Settings → General → Use the WSL 2 based engine
  • 防火墙：Windows Defender 防火墙可能阻止容器端口，需添加入站规则或临时关闭
  • 路径问题：Windows 路径在 docker-compose.yml 中使用 ./ 相对路径，或使用 /c/Users/... 格式
  • 换行符：配置文件和脚本使用 LF 而非 CRLF，可用 VS Code 右下角切换
网络拉取慢 — 配置镜像加速器
Docker Desktop 配置镜像加速# Docker Desktop → Settings → Docker Engine # 添加以下配置： { "registry-mirrors": [ "https://docker.1ms.run", "https://docker.xuanyuan.me" ] }
  • 配置后点击 Apply & Restart 重启 Docker Desktop
  • 验证加速是否生效：docker info | findstr "Registry Mirrors"
  • 如果镜像加速也慢，可尝试使用代理或在网络条件好的时候预拉取镜像

Kafka 常见问题
  • 连接超时：确保 KAFKA_ADVERTISED_LISTENERS 设为 PLAINTEXT://localhost:9092，不是容器内部地址
  • Topic 自动创建失败：检查 KAFKA_AUTO_CREATE_TOPICS_ENABLE 是否为 true
  • 消费不到消息：确认 --from-beginning 参数，或检查 consumer group offset
  • Zookeeper 未就绪：Kafka 启动依赖 Zookeeper，确保 depends_on 配置正确
排查 Kafka 连通性# 检查 Kafka broker 状态 docker exec bigdata-kafka kafka-broker-api-versions ^ --bootstrap-server localhost:9092 # 检查 Zookeeper 连接 docker exec bigdata-zookeeper echo ruok | nc localhost 2181
