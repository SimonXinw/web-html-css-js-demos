岗位背景
电商独立站项目： 基于 Next.js SSR 架构开发前端 +  Shopify 平台 API（GraphQL/JSON）以及管理后台。
核心目标： 品牌站整站动画扎实的动画设计，动画开发，动画性能，网站性能优化，动画性能优化，并打造具备"Apple 官网级"视觉冲击力的品牌交互体验。

岗位职责
1. 使用 Next.js、React SSR 框架开发和维护电商独立站，对接 Shopify Headless API。
2. 负责品牌核心页面的高保真还原，能够通过代码精确实现 Apple 风格的 Scrolllytelling（滚动叙事）、视差滚动及复杂的转场动画。
3. 针对复杂动效进行深度性能调优，确保在多端（尤其是移动端）保持 60FPS/120FPS 的流畅度，严格把控 LCP、CLS 等关键 Web  指标。
4. 基于 React 封装可复用的高性能动画组件库，提升品牌站点的整体动效一致性与开发效率。
5. 与产品及 UI 深度合作，完成需求分析、技术规划；编写流程图及相关技术文档。
6. 确保高质量交付，保证在不同设备与浏览器上的良好兼容性。

面试题
第一题：手写一个 LERP（线性插值）平滑跟随效果
题目： “请在白板或代码编辑器中，实现一个简单的‘平滑跟随鼠标’的圆形。鼠标移动时，圆形不是瞬间到达，而是有一种‘带阻尼的平滑过渡’效果。请简述其原理。”
● 看似考察： 简单的鼠标事件。
● 实际考察：
  ○ 数学基础： 是否知道 current + (target - current) * easing 这个核心公式？
  ○ 渲染机制： 是否知道必须使用 requestAnimationFrame 而不是 setTimeout？
  ○ 性能优化： 是否知道使用 transform: translate3d 而不是修改 top/left？是否知道为什么要开启 GPU 加速？
  ○ 代码习惯： 是否在组件销毁时正确清理了动画帧，防止内存泄漏？

第二题：解决 Next.js 中的 “FOUC” 动效闪烁问题
题目： “在 Next.js 的 SSR 模式下，当你使用 GSAP 或 Framer Motion 给一个标题设置 opacity: 0 的初始状态，并在进入页面时执行淡入动画，用户有时会先看到标题闪烁一下（即先看到 100% 透明度再变回 0% 开始动画）。你如何从根本上解决这个问题？”
● 看似考察： 一个小 Bug。
● 实际考察：
  ○ SSR 深度理解： 是否理解 HTML 在服务器生成，而动画脚本在客户端运行的时间差？
  ○ React 生命周期： 对 useLayoutEffect 与 useEffect 差异的理解（在渲染前锁定样式）。
  ○ CSS 配合： 是否懂得利用 CSS 变量或内联样式在 HTML 结构中预设状态，而不是完全依赖 JS 初始化。
  ○ GSAP 专有技术： 是否知道 gsap.context() 或 gsap.set() 的正确用法。

第三题：拆解 Apple 风格的“滚动序列帧”
题目： “Apple 官网经常有一种效果：随着页面滚动，一个产品（如 MacBook）会逐渐翻开。如果是你来落地，你会选择用 <video> 标签控制进度，还是用 <canvas> 绘制图片序列？为什么？”
● 看似考察： 技术选型。
● 实际考察：
  ○ 多端兼容性： 是否知道移动端浏览器（尤其是 iOS Safari）对视频自动播放和滚动控制的严格限制（低电量模式卡顿、黑屏问题）。
  ○ 性能瓶颈：canvas 绘制序列帧的内存压力（100张高清图如何预加载？如何防止崩溃？）。
  ○ 数学逻辑： 如何将滚动的像素值（ScrollTop）精准映射到图片的索引（Index）上。
  ○ 交互体验： 如何处理滚动惯性（Smooth Scrolling）与动画帧的同步。

第四题：SVG 路径动画的“黑科技”
题目： “现在有一个不规则的品牌 Logo 路径（SVG Path），要求实现：随着用户向下滚动页面，Logo 像被一支笔画出来一样逐渐呈现。请问最核心的两个 CSS 属性（或 GSAP 属性）是什么？如果路径极其复杂，如何获取路径的总长度？”
● 看似考察： 属性记忆。
● 实际考察：
  ○ 核心原理：stroke-dasharray 和 stroke-dashoffset 的骚操作。
  ○ API 掌握： 是否知道浏览器原生的 path.getTotalLength() 方法。
  ○ 实战经验： 是否遇到过 SVG 导出时坐标轴不对位的问题？如何处理多条路径同时画出的节奏感（Stagger 错开效果）？

💡 综合观察点（面试官参考）：
在对方回答时，您可以重点观察他是否提到以下高频词汇：
1. Composite Layer (合成层)： 知道优化动画不触发重排（Reflow）。
2. Will-change： 知道如何提前告诉浏览器分配资源。
3. Lerp / Spring (弹簧物理)： 知道动画要有物理感，而不是死板的线性移动。
4. Batching (批处理)： 在 Shopify 多产品列表页，如何批量初始化动画而不卡顿。
