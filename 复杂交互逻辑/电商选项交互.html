核心就两件事：① 每个按钮的状态怎么算（computeOptionState）；② 当前选的是哪一条变体（resolveVariant）。

1. 术语对照（和页面、代码一致）

术语 含义 在代码里
选项层 / Option 商品的一个维度，如「颜色」「尺码」。第 0、1、2… 层顺序固定。 optionDefinitions[]，含 name、values[]
变体 / Variant 每层各选一个值后对应的一条 SKU（独立价格、库存等）。 variants[]，optionValues 下标与层号一致
当前选择 selection 用户当前点了什么；某层未选为 null。 selection["颜色"] 等
availableForSale 是否允许下单（可接库存、上架状态）。 true 时「加入购物车」才可点
available 本 Demo 辅助字段，用于按钮文案区分「暂时不可售 / 已售罄」等。 见 updateSummary

2. 数据模型（为什么要两套数组）

2.1 optionDefinitions —— 页面上画哪些按钮

声明有几层、每层叫什么、每层有哪些展示用候选值。注意：界面上可以画出某个值，但该值可能对当前上层组合无变体，这时由算法标成禁用，而不是从数组里删掉。

2.2 variants —— 真正可卖的「行」

后台通常一行 SKU 一条。前端用这些行判断：某组合是否存在、能不能卖。

举例（双层）：层0=颜色，层1=尺码。变体 ["黑色","S"] 是「黑+S」这一条 SKU。 若数据里没有 ["黑色","XL"]，用户选「黑色」后，尺码 XL 应算 disabled（根本没有这条组合，不是卖光）。

3. 三种 UI 状态（务必分清业务含义）

状态 含义（一句话） 本页表现
available 在当前上层选择下，存在含该值的变体，且至少一条可售。 正常按钮；选满后一般可加购。
sold_out 组合存在，但匹配到的变体全部不可售。 黄底 +「售罄」；可选中加购禁用。
disabled 当前上层下没有任何变体用到该值。 灰、删除线、不可点。

易混点：「售罄」= 有 SKU 但不能卖；「禁用」= 没有这条 SKU。两种含义不同，样式应区分。

4. 核心算法：computeOptionState(layerIndex, value)

问题：画第 i 层、值为 v 的按钮时，应是 available / sold_out / disabled 哪一种？

4.1 与源码一致的步骤

  1. 取出全部 variants。
  2. 按上层已选过滤：对层 0 … i-1，若 selection[该层名] 已是具体值，则只保留 variant.optionValues[层号] === 该值；若某层为 null，该层不参与过滤。
  3. 再过滤：只保留 variant.optionValues[i] === v，得到 matches。
  4. matches.length === 0 → disabled。
  5. 否则若存在 availableForSale === true → available。
  6. 否则 → sold_out。

4.2 手工推演（双层，已选 颜色=黑色）

问：尺码 XL？ 若无任何变体是「黑+XL」→ 过滤后为空 → disabled。

问：尺码 L？ 若仅一条「黑+L」且 availableForSale: false → sold_out。

第 0 层：没有「上层」，步骤 2 里循环 0…-1 不执行，只看「第 0 层 = v」的所有变体。

5. 当前变体：resolveVariant()

当每一层在 selection 里都有值时，按层顺序拼成元组，在 variants 里找 optionValues 完全一致的一行；用于价格、SKU、加购。未选满或找不到则清空展示。

6. 改选上层：clearLowerLayers + autoFillFromLayer

6.1 为何要 clear

例如从「灰+M」改成「藏青」，若不存在「藏青+M」，下层「M」在新状态下会变成 disabled，必须把非法下层清掉（或改掉），否则无法和任何变体对齐。

6.2 为何要 autoFill

清完后若让用户再逐层点，体验碎。因此在用户点完第 k 层后，对 k+1、k+2… 每层依次选：第一个 available，否则第一个 sold_out。切换案例 Tab 时用 autoFillFromLayer(-1) 从第 0 层起填满一整条默认规格链。

7. 脚本函数索引（对照学习）

computeOptionState(i, v) — 第 i 层、值 v 的三态
resolveVariant() — 选满后解析当前 SKU
clearLowerLayers(k) — 改第 k 层后清除变非法的下层
autoFillFromLayer(k) — 从 k+1 层起自动「先可售后售罄」
renderOptions() — 生成选项 DOM
updateSummary() — 价格、SKU、加购按钮
switchCase(key) — 切换 DEMO_CASES 并重置默认选中

8. 与 Shopify 真机的差异

  • Shopify 另有：?variant=、默认变体、多市场、主图随变体切换等，本页未实现。
  • 真实接口字段可能是 option1 / selectedOptions 等，但过滤与三态逻辑与本文一致。

9. 三个 Tab 各练什么

案例 学习重点
单层 无上层过滤；区分 XL「无变体=禁用」与 L「有变体不可售=售罄」。
双层 典型服装流；下层状态随上层变化。
三层 同一套代码扩展到三维 optionValues；如「白+S」无整条变体时 S 被禁用。
